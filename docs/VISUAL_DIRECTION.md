# 디자이너가 만드는 세상 — Visual Direction 0.1

이 문서는 앞으로 생성되는 모든 게임 이미지와 UI의 단일 시각 기준이다.
기능이 추가될 때마다 이 문서와 함께 버전을 올린다.

## 1. 세계의 출발점

- 게임은 `빈 아이소메트릭 바닥 + 접속한 디자이너 한 명`에서 시작한다.
- 건물, 퀘스트, 화폐, 상점, 장식은 처음부터 존재하지 않는다.
- 새 요소는 실제 사용자 요구를 반영한 하나의 커밋으로만 세계에 들어온다.
- 화면이 비어 있는 상태 자체가 미완성이 아니라 게임의 핵심 장면이다.

## 2. 기준 이미지

시각적 원본은 `public/assets/ui/designer-world-login-v1.png`다.

이 이미지에서 이어받아야 하는 특징:

- 고해상도 2.5D 아이소메트릭 픽셀아트
- 깊은 웜 차콜 설계 보드
- 매트 콘크리트, 미완성 목재, 브러시드 메탈
- 본 화이트와 낮은 채도의 세이지
- 한 번에 하나만 사용하는 뮤트 코랄 포인트
- 저녁 작업실처럼 차분하고 집중된 조명
- 귀여운 농장 게임보다 독립 디자인 매거진에 가까운 태도

금지:

- 보라·파랑 네온, 사이버펑크 광원
- 판타지 RPG 장식
- 과도한 아기자기함과 둥근 장난감 질감
- 매끈한 벡터풍, 광택이 강한 3D 렌더링
- 외부 글로와 과포화 색상

## 3. 캐릭터 원칙

첫 캐릭터 `designer-01`이 앞으로의 인체 비율과 재질 기준이다.

- 현실적인 젊은 디자이너의 옷차림
- 작은 화면에서도 읽히는 차콜 실루엣
- 셔츠는 본 화이트, 도구 하나만 코랄 포인트
- 정면보다 아이소메트릭 월드와 맞는 3/4 방향
- 얼굴보다 작업 도구와 자세로 직업을 표현
- 캐릭터별 성격은 바꾸되 픽셀 밀도와 신체 비율은 유지

다음 캐릭터 단계:

1. 현재 정지 원형 승인
2. 4방향 정지 포즈
3. 4방향 걷기 프레임
4. 의상과 피부·머리 커스터마이징
5. 네트워크에서 여러 캐릭터 동시 표시

## 4. 이미지 생성 계보

모든 새 이미지 생성은 다음 절차를 따른다.

1. 로그인 기준 이미지를 스타일 레퍼런스로 첨부한다.
2. 자산의 역할, 방향, 크기, 팔레트, 금지 사항을 프롬프트에 명시한다.
3. 생성 프롬프트를 `art_sources/<category>/`에 저장한다.
4. 게임용 최종 PNG만 `public/assets/<category>/`에 둔다.
5. 파일명은 `<subject>-<state>-v<number>.png`로 버전 관리한다.
6. 승인된 자산을 조용히 덮어쓰지 않는다.

## 5. UI와 Liquid Glass

픽셀아트는 세계의 콘텐츠 레이어다. Liquid Glass는 기능 레이어다.

- 로그인 폼, 접속 상태, 버튼, 입력, 향후 메뉴에만 유리를 사용한다.
- 유리는 어두운 월드가 비칠 정도의 투명도와 1px 내부 굴절선을 가진다.
- 긴 글과 게임 바닥을 반투명 유리 카드로 덮지 않는다.
- 버튼은 최소 44px, 모바일 입력 텍스트는 16px 이상이다.
- 선택과 주요 행동에만 코랄을 사용한다.
- 투명도 감소와 모션 감소 환경에서는 불투명한 차콜 표면으로 전환한다.

## 6. 현재 화면 계약

로그인:

- 이름 한 칸과 진입 버튼 하나만 제공한다.
- 가입 탭, 계정 팝업, 공지 팝업, 보상 팝업을 만들지 않는다.
- 로그인 키아트가 게임 전체의 시각적 약속을 전달한다.

월드:

- 빈 바닥, 캐릭터, 접속 상태, 나가기만 존재한다.
- 바닥 클릭과 방향키 이동은 가능하다.
- 가짜 온라인 인원이나 구현되지 않은 멀티플레이 상태를 표시하지 않는다.
