> **UI 규격 이전 안내 (2026-07-23)**
> 버튼·입력·HUD·팝업·내비게이션·타이포그래피·Liquid Glass 재질의 SSOT는
> [`docs/DESIGN_SYSTEM.md`](./DESIGN_SYSTEM.md)다. 이 문서의 기존 §2 나무 프레임 규칙은
> 레거시 구현 기록이며, 신규 UI와 리뉴얼 화면에는 적용하지 않는다.

> 이 문서는 홍대마을의 **단일 진실 원천(SSOT)** 이다. 모든 신규 아트·UI·카피·사운드·
> AI 생성물은 여기 정의된 규격을 **반드시** 따른다. 규격을 바꿔야 하면 이 문서를 먼저
> 고치고, 코드/에셋을 맞춘다. "항상 같은 스타일"이 목표다.
>
> 우선순위: 사용자 지시 > **이 문서** > 개별 파일 관행 > 기본값.

## 0. 한 줄 컨셉

**서울 홍대입구, 차분하고 아기자기한 웜톤 아이소메트릭 픽셀 코지 마을.** 주인공 건물은
**마인드 포레스트(인싸이트) 빌딩**. 생활감 높은 디오라마 공간에서 꾸미기·수집·소셜을 즐긴다.

### 0.1 비주얼 목표

- 품질 기준: `Whisper of the House`처럼 생활 소품이 촘촘하고 내부가 읽히는 2.5D 디오라마.
- 참고 작품의 자산·구도를 복제하지 않고 **홍대 골목·한국 상점·회사 생활**로 독자화한다.
- 현재 직교형 게임은 기능 보존 레이어다. 아이소메트릭 수직 슬라이스를 검증한 뒤 씬별로 전환한다.
- 전환 중에도 논리 좌표·충돌·네트워크·경제는 유지하고 렌더링 계층만 교체한다.

## 1. 아트 원칙

- **픽셀아트 only.** 벡터·사진·3D 렌더 금지. `pixelArt: true`, `image-rendering: pixelated`.
- **무드**: 차분한 웜톤(따뜻한 갈색·크림·세이지). 채도 낮게, 대비 부드럽게. 네온·형광 금지.
- **프로시저럴 우선 + AI 보강**: 모든 아트는 코드 프로시저럴로 폴백이 존재해야 한다.
  AI 스프라이트는 있으면 얹고, 없어도 게임이 동일하게 돌아가야 한다(404 안전).

### 1.0a 타이틀 키아트

- 데스크톱은 `title_keyart_v2` 16:9, 모바일은 `title_keyart_mobile_v2` 9:16 전용 구도를 사용한다.
  모바일을 데스크톱 중앙 크롭으로 대체하지 않는다.
- 키아트는 실제 플레이의 집 내부·아틀리에·펫·옥상 정원·공동 장소를 한 아이소메트릭 공간 안에
  배치한다. 게임에 없는 대형 랜드마크나 전투 장면으로 콘텐츠를 과장하지 않는다.
- 런타임은 WebP, 수정·재압축용 마스터는 PNG다. 생성 기록과 프롬프트는
  `art_sources/ui/title_keyart_v2.prompt.md`에 보존한다.

### 1.0 아이소메트릭 공간 규격 (`world/isometric.ts`)

- 논리 타일은 기존 **32px**를 유지하고 화면 타일만 **64×32px(2:1 다이아몬드)**로 투영한다.
- 서버·충돌·퀘스트 좌표는 직교 논리 좌표가 SSOT다. 저장 데이터에 화면 좌표를 넣지 않는다.
- 높이 1단위는 16px. 건물·벽·가구 높이는 정수/0.5 단위 elevation으로 관리한다.
- depth는 오브젝트 풋프린트의 가장 남쪽 모서리(`tx+w+ty+h`)를 기준으로 계산한다.
- 클릭·배치는 화면→아이소메트릭 역투영→논리 타일 순서로 처리한다.
- WASD는 화면 방향 기준으로 느껴지도록 입력을 논리 격자 축으로 회전한다.
- 검증용 수직 슬라이스는 개발 서버의 `?iso&offline`으로 진입한다.

### 1.0a 골목 권역과 여권 (`world/neighborhoodDistricts.ts`)

- 34×24 논리 월드는 옥상 주택가·카페 골목·중앙 광장·살림 창작골목·동행자 거리·역 앞 산책문·
  동쪽 외곽숲의 7개 권역으로 빠짐없이 분류한다. 권역 판정과 이름·색·안내 목표는 이 모듈이 SSOT다.
- 발견은 플레이어가 실제 논리 좌표에 들어온 순간 사용자별 영구 도장으로 저장하며 재진입으로
  중복 증가하지 않는다. 저장본은 미등록·중복 ID와 잠긴 최애 전시를 정규화한다.
- 미발견 권역도 길 안내를 막지 않는다. 각 `guideTarget`은 자기 권역의 통행 가능 타일이어야 하고
  기본 스폰에서 기존 충돌 회피 경로 탐색으로 도달할 수 있어야 한다.
- 지도는 월드 축소판과 여권을 한 화면에 보여주고, 발견한 권역만 최애 동네로 전시한다. 발견 수·
  안전 권역 발견 수·최애 전시는 퀘스트 지표와 같은 진행 새로고침 주기에서 동기화한다.

### 1.0b 전역 마을 찾기 (`guidance/villageSearch.ts`)

- 메뉴·월드 활동·주민·권역·퀘스트의 기존 SSOT에서 색인을 만들고, 검색 전용으로 목적지 좌표나
  퀘스트 정의를 복제하지 않는다. 신규 활동과 퀘스트는 자동으로 색인에 들어와야 한다.
- 한국어 생활 별칭과 띄어쓰기 차이를 허용하되 검색어 토큰은 모두 일치해야 한다. 정확한 제목,
  실제 장소·주민·권역, 진행 중 퀘스트, 완료·잠금 항목 순으로 결과 우선도를 계산한다.
- 결과 선택은 기존 패널 열기 또는 충돌 회피 금빛 경로만 사용한다. 주민은 실제 배치 좌표,
  활동은 `ISO_VILLAGE_ACTIVITIES`, 권역은 `guideTarget`, 퀘스트는 공통 metric 목적지로 연결한다.
- 빈 검색에는 여덟 개의 가벼운 시작점만 보여 주고, 모바일은 빠른 검색어와 세로 결과 목록으로
  접는다. 키보드 `F`와 하단 `찾기`를 동등한 진입점으로 제공한다.

### 골목 동아리 응원 키트와 아지트

- 동아리 장 발간은 별도 화폐나 능력치를 만들지 않고, 같은 순서의 응원 키트와 2.5D 아지트 장식을
  한 점씩 영구 해금한다. 이전 저장의 연속 발간 기록은 `v2`로 무손실 이관한다.
- `pinnedClubId`는 하나의 최애 동아리, `featuredClubIds`는 최대 세 곳의 대표 아지트를 뜻한다.
  대표 전시를 바꾸거나 내려도 발간·키트·장면 재생 기록은 줄지 않는다.
- 아지트 장면은 현재 저장 캐릭터와 활성 펫을 공통 픽셀 렌더러로 합성한다. 방 좌표나 외형 복제본을
  저장하지 않으며, 동아리별 색·소품과 발간 단계만 결정적으로 파생한다.
- 발간 30장, 키트 30점, 완성 키트 6종, 대표 전시 3곳, 장면 재생은 공통 퀘스트 지표와 동아리
  활동 지점 길 안내를 공유한다. 기간 제한·순위·전투 효과는 두지 않는다.

### 첫 생활 멘토 연대기

- 새 이웃 취향 나침반의 환영 소포를 받은 뒤에만 같은 방향의 3장 멘토 연대기가 열린다. 소포를
  받지 않았다는 이유로 다른 방향이나 기존 활동을 잠그지 않는다.
- 여섯 방향은 각각 3장, 각 장은 기존 평생 지표 2개를 읽는다. 장은 순서대로 보존하지만 요구
  활동은 먼저 해도 소급 인정하며 실패·기한·재시작 상태를 만들지 않는다.
- 장면은 현재 캐릭터와 활성 펫을 공통 렌더러로 합성한다. 저장에는 장 ID·대표 3장·재생 횟수만
  남기고 외형·위치·전투 수치의 복제본을 넣지 않는다.
- 18장 보존, 6루트 완주, 대표 3장, 재생 횟수는 모험 일지·골목 탐험 숙련·취향 프로필·마을
  여정이 같은 지표를 읽는다. 대표 변경과 재생은 능력치나 보상 확률을 바꾸지 않는다.

### 1.0c 골목 퀘스트 신호 (`guidance/worldQuestSignals.ts`)

- 월드 표식은 `◆ 추적 → ✓ 보상 → ↗ 진행 → 새 이야기` 순으로 우선한다. 일반적인 `!/?`를 복제하지
  않고 한국어 상태를 함께 적어 색상에만 의존하지 않는다.
- 잠긴 목표, 완료한 영구 목표, 수령한 일일 목표는 숨긴다. 진행이 없는 수집·숙련 목표는 모험 일지에만
  남겨 풍경을 가리지 않으며, 장소별 한 건·활동 8곳·주민 2명으로 제한한다.
- 퀘스트 정의의 `registryKey`를 공통 목적지 규칙에 연결하고, 주민별 신뢰 지표는 해당 주민 이름표에
  고정한다. 지도·명함처럼 패널 안에서 이어지는 기록은 중앙 모험 일지 한 곳으로 모은다.
- 표식을 누르면 목표 추적과 기존 충돌 회피 금빛 길 안내를 재사용한다. `✓ 보상`은 완료 활동을 반복시키지
  않고 모험 일지의 오늘 탭을 연다.

### 1.0d 골목 기록 도착 (`guidance/questMilestones.ts`)

- 완료 피드백은 전체 화면을 멈추는 팝업이 아니라 액션 바 위의 비차단형 픽셀 기록 카드로 보여 준다.
  첫걸음·오늘·이야기·수집·숙련은 서로 다른 도장색과 글자 표식을 사용하고 색에만 의존하지 않는다.
- 한 행동으로 여러 목표가 끝나면 `첫걸음 → 이야기 → 오늘 → 수집 → 숙련` 순으로 대표 한 건을 고르고,
  나머지는 같은 카드의 `함께 기록됨`에 요약한다. 알림 여러 장을 연속 재생하지 않는다.
- 사용자별 수신 이력을 영구 보존한다. 첫 실행에서는 현재 완료 기록을 기준선으로 삼아 과거 알림을
  쏟지 않고, 집 등 다른 씬에서 새로 끝낸 목표는 아이소메트릭 마을 복귀 시 한 번만 보여 준다.
- 직접 선행 관계가 있는 다음 퀘스트만 `NEXT STORY`로 표시한다. 버튼은 그 목표를 실제 HUD 추적으로
  바꾼 뒤 모험 일지를 열며, 일일 완료는 오늘 탭의 보상 수령으로 곧장 이어진다.

### 1.0e 골목 이야기 대화 (`residents/residentQuestDialogue.ts`)

- 주민별 퀘스트는 별도의 수락·거절 저장 상태를 만들지 않는다. 잠금이 풀린 순간 이미 열린 이야기이며,
  기존 지표 진행을 그대로 읽어 과거 행동도 자동 반영한다.
- 10명 모두 직업·관계 서사에 맞춘 고유 말투를 사용한다. 대화는 `새 이야기·진행 중·함께 추적`을
  문장과 영문 도장으로 함께 구분하고, 기한·실패·진행 소실이 없다는 안심 문구를 항상 제공한다.
- 주민 본인 클릭과 이름표 위 퀘스트 신호는 같은 픽셀 대화 패널로 들어간다. 해당 주민에게 표시할
  퀘스트가 없으면 일반 메뉴가 아니라 그 주민의 관계 수첩을 정확히 선택해 연다.
- `이 이야기 따라가기`는 기존 `focusedQuestId`만 바꾸며 퀘스트를 소비하거나 재시작하지 않는다.
  `주민 수첩 보기`는 같은 주민의 이야기 탭으로 연결하고, 닫기는 어떤 기록도 변경하지 않는다.

### 1.0f 골목 비밀 기록 (`guidance/alleySecrets.ts`)

- 환경 단서는 4편×3개, 총 12개다. 모두 실제 통행 가능한 논리 타일에 두고 기본 스폰에서 충돌 회피
  경로로 도달할 수 있어야 한다. 활동 타일을 재사용한 숫자 버튼이 아니라 월드의 작은 ◇ 흔적으로 렌더한다.
- 미발견 흔적은 가까운 거리에서만 보여 풍경과 퀘스트 표식을 가리지 않는다. 멀리서 수첩의 길 안내를
  선택하면 순간이동 없이 금빛 발자국을 만들고, 목표 타일 도착 시 해당 단서를 자동 기록한다.
- 세 단서는 순서 없이 발견한다. 한 편을 완결하면 마지막 문장과 픽셀 삽화를 영구 보존하고, 완결한
  이야기만 최애로 전시할 수 있다. 최애를 바꾸거나 내려도 단서·완결·퀘스트 진행은 줄지 않는다.
- 외곽숲 단서는 관찰 모드에서도 안전하게 발견할 수 있어야 한다. 전체 시스템에 기간 한정, 실패,
  일일 횟수, 전투 강제, 경쟁 순위는 두지 않는다.
- 지도·전역 찾기·`L`을 동등한 기록책 진입점으로 제공한다. 발견 수·완결 수·최애 전시는 퀘스트
  지표와 같은 진행 새로고침 주기에서 동기화한다.

### 1.1 게임 아트 팔레트 (SSOT = `src/game/art/palette.ts`)
프로시저럴 지형·건물·캐릭터·방은 **`PAL` / `ROOM_PAL` 안에서만** 색을 고른다. 새 색이
필요하면 팔레트에 먼저 추가한다. 대표값:
- 바닥: `alley 0x8a7d6e`, `grass 0x7d9468`, `plaza 0x9a8d7c`, `road 0x5c5852`
- 건물: `wall 0xc4a98a~0x8fa38a`, `roof 0x6e5c4c`, `door 0x8a5a3a`, `winGlassWarm 0xf2d8a0`
- 캐릭터: `skin 0xf0cfae`, `outline 0x3a2f26`
- 방: `floorWood 0xa8896a`, `wallPaper 0xd8cbb6`

### 1.2 캐릭터 규격 (`src/game/art/characterArt.ts`)
- 현재 캐릭터는 네이티브 32×48px, 4방향 × 4프레임 시트이며 걷기 애니는 `[1,2,3,2]`다.
- `characterArtHd.ts`가 월드·NPC·멀티플레이·사진·프로필 미리보기의 단일 렌더 계약이다.
  다음 확장은 저장 형식을 바꾸지 않고 8방향 및 action 프레임을 추가하는 방식으로 진행한다.
- 외형 조합: 피부·헤어·염색·눈매·입·얼굴 포인트·상의/하의 실루엣·상의 무늬·양말·신발·안경·모자·등 장식의
  독립 레이어로 현재 `619,173,642,240,000`가지 조합을 만든다(`appearance.ts`).
  작은 프레임에서도 눈·무늬·가방 실루엣이 서로 구분되어야 하며 아치형·비율 왜곡은 금지한다.
- 레이어 순서: 등 장식→다리/양말/신발→몸통/무늬→팔→머리/얼굴→헤어→**안경**(눈 위, 뒤통수 제외)→**모자**(정수리).
  안경·모자는 옵션(0=없음)이며 레거시 저장값엔 없어도 0으로 정규화된다.
- 커스터마이즈 패널은 **큰 픽셀 미리보기 필수**(`paintCharacterFrame`)로 방향 회전·즉시 반영.
- 신규 외형 필드는 모두 optional 저장 필드로 추가하고 `normalizeAppearance`에서 0으로 보정해 기존 계정·presence·옷장을 유지한다.
- 눈매·입·얼굴 포인트·상의 무늬·양말은 `styleCatalog.ts`가 해금 SSOT다. 기본 18종은 즉시 열고
  희귀 10종은 기존 모험 퀘스트의 배지로 연다. 잠긴 항목은 화살표 순환과 랜덤 코디에서 건너뛰며,
  도감에는 필요한 배지와 획득 퀘스트를 함께 표시한다. 레거시 코디의 잠긴 값은 무손실 보존한다.
- 희귀 스타일 진행도는 `rare_styles` 최대값 퀘스트 지표로 동기화한다. 3종 해금은 이야기 목표,
  10종 완성은 수집 목표가 되어 커스터마이징이 장기 수집 콘텐츠로 이어져야 한다.
- origin `(0.5, 0.78)` — y=46 발바닥이 충돌 박스 바닥에 오게. depth: 플레이어 10, NPC 9.

### 1.2b 펫 규격 (`src/game/art/petArt.ts` / `pets/`)
- 20×20px, **우향 1프레임**(좌향은 스프라이트 flipX). 종별 귀(`earType`)·꼬리(`tail`)·색
  파라미터로 실루엣을 구분하고 상단·목·몸통 여백에 액세서리 레이어를 그린다. 캐릭터와 동일한
  아웃라인·볼터치 감성을 유지한다.
- 팔로워는 위치 이력 버퍼로 몇 프레임 뒤처져 따라오고 멈추면 바운스(동물의 숲 감성).
  depth 9(플레이어 아래). 동행 상태·보유는 `PetStore`(로컬) + 서버 `owned_pets`(0011) 병합.
- 가격 SSOT = 서버 `adopt_pet` RPC. `pets.ts` 가격과 SQL을 **항상 동일하게** 유지.
- 친밀도(0~100)·성장 4단계. 쓰다듬기(클릭)·먹이(하루 1회)로 상승, 단짝(2+)이면 머리 위 하트.
- **선물 루프**: 데리고 걷다 보면 가끔 보물 조각을 물어다 준다(`petGift.ts` 순수 로직 —
  친밀도↑ → 자주·후하게, 최소 18s). 코인은 서버 권한이라 조각(로컬)으로 보상.

### 1.2c 외곽숲 모험 보호 (`battle/adventureComfort.ts`)

- 새 사용자와 손상된 저장본은 항상 `peaceful` 관찰 모드로 시작한다. 외곽숲 경계를 밟는 것만으로
  몬스터가 추적·공격하거나 플레이어가 자동 공격해서는 안 된다.
- `expedition` 원정 모드는 외곽숲 상단 HUD에서 사용자가 직접 켠 뒤에만 작동한다. 같은 자리에서
  즉시 관찰 모드로 돌아갈 수 있고, 이때 처치 연속 기록만 종료하며 장비·레벨·도감은 유지한다.
- 관찰 중에는 외곽숲 안에서도 안전 생활권과 같은 회복 규칙을 적용한다. 모드 버튼은 데스크톱과
  모바일 모두 현재 상태·결과를 글자로 표시하고 색상만으로 안전 여부를 전달하지 않는다.
- 최초 원정 선택과 최초 안전 복귀는 사용자별 영구 기록으로 보존하고, 각각 이야기 퀘스트에 소급
  반영한다. 개발 검증용 `?iso&offline&hunt`만 원정 모드를 자동으로 켤 수 있다.
- **개성 프로필**: 펫별 별명(공백 포함 8자)·성격 6종·액세서리 7종을 기록한다. 성격은 능력치에
  불이익을 주지 않고 언제든 변경할 수 있다. 액세서리는 친밀도·놀이·트릭·연습·추억의 실제 기록으로
  열리며 잠금 카드에 현재값/목표값을 함께 표시한다.
- 장착 장식은 `ensurePet(species, accessory)` 텍스처 키와 DOM 미리보기에 동일하게 반영한다.
  사용자 별명과 장식 id는 정제 후 Realtime presence에 전달해 온라인 이웃에게도 같은 펫이 보인다.
  기존 `hv-pets-v1` 저장에는 optional 필드로 병합해 레거시 친밀도·소유·트릭을 잃지 않는다.
- 순수 계산(위치·선물)은 Phaser 비의존 모듈로 분리해 유닛 테스트한다(`petFollowMath`·`petGift`).
- **동행 산책 수첩**: 8코스마다 1·3·6 기록에 기념품을 하나씩 배치해 24칸 도감을 만든다. 성격이
  코스의 선호 성격과 맞으면 2칸, 아니어도 1칸이 올라 실패·감소가 없다. 도감은 펫별 기록 중 코스별
  최고점을 사용하고, 완주 뒤에도 다른 펫과 같은 길을 다시 걸어 동행 조합을 늘릴 수 있다.
- 산책 장면은 `petOutingArt.ts`에서 480×270 절차형 픽셀 레이어로 그리며 `paintPet`을 합성해 실제
  별명·성격·장식과 같은 동행 펫을 보여 준다. 데스크톱은 큰 풍경과 세로 코스 색인, 모바일은 단일 열이다.

### 1.2c 몬스터·전투 규격 (`src/game/battle/`, `monsterArt.ts`)
- 몬스터 16×16, `shape`(slime/horned/spiky/ghost/winged/golem)+팔레트로 30종 구분.
  높은 티어일수록 어둡고 성난 표정(눈썹·이빨). 사냥터(경의선 숲길)에서만 출몰.
- **전투는 근접 자동**: 가까이 가면 내가 자동으로 휘두르고(✦), 몬스터도 근접 시 반격(붉은 피격).
  HP 바는 몬스터 위. 처치 시 💥 + 경험치·조각.
- **레벨/경험치 곡선은 가파르게**(`combat.ts` `level^1.7`) — 너무 쉽지 않게. 무기(코인)로 공격력 보강.
- **사망 = 경험치 감소 + 마을 부활 + 피로(15s, 이동·공격 절반)**. 순수 로직은 `combat.ts`로 분리해 테스트.
- 가격 SSOT = 서버 `buy_weapon` RPC(0012). `weapons.ts`와 SQL 가격을 **항상 동일하게** 유지.
- 마을은 평화 유지: 몬스터는 필드에 가둠. 필드에 들어온 행인은 겁먹고 도망(`npcAmbient` flee).
- **장착 무기는 캐릭터 손에 보인다**(`weaponArt.ts` 검 스프라이트, 방향별 배치·스윙 연출).
  이름표는 `● 닉 Lv.N`, 그 위에 레벨 **호칭**(`titles.ts`, 예: 견습 사냥꾼→전설의 영웅).

### 1.2d 생활 숙련 규격 (`src/game/progression/lifeMastery.ts`)

- 전투 레벨과 별도로 **골목 탐험·패션 감각·공간 꾸미기·동행 돌봄·이웃 관계·골목 생활·도시 정원·골목 요리·물정원 낚시·외곽 모험**
  10분야를 Lv.1~10으로 기록한다. 한 분야만 강제하지 않고 모든 생활 콘텐츠가 성장 경로가 된다.
- 경험치는 `QuestState.lifetimeCounts`에서 결정적으로 파생한다. 퀘스트 진행 표시는 목표치에서 멈추지만
  평생 활동 원본은 계속 누적해, 완료 이후의 반복 플레이도 숙련에 반영한다.
- 각 분야 카드는 현재 등급·레벨 XP·다음 추천 행동·정확한 위치·목표 진행을 함께 보여야 한다.
  초보자는 무엇을 할지 바로 이해하고, 수집형 이용자는 전 분야 Lv.10과 합계 100을 장기 목표로 삼는다.
- 분야 Lv.5, 전 분야 Lv.3, 합계 Lv.45는 실제 장착형 휘장 퀘스트로 연결한다. 숙련 파생 지표는
  다시 경험치 입력으로 사용하지 않아 반복 동기화 시 값이 불어나는 순환을 금지한다.

### 1.2d-a 생활 시너지 덱 (`lifeSpecialty.ts`, `lifeSynergies.ts`)

- 대표 전문성 세 장을 MMORPG의 빌드 조합처럼 사용하되 공격력·재화·드롭률 보너스는 주지 않는다.
  서로 다른 세 생활 분야의 정확한 조합으로 고유 호칭·서술·색·픽셀 별자리 카드·기념 표식을 발견한다.
- 큐레이션한 조합은 18종이며 분야 서명이 중복되면 안 된다. 현재 선택이 완성 조합이 아니면 겹치는
  분야가 많은 미발견 조합부터 세 개 제안해 정답을 외우지 않아도 탐색할 수 있게 한다.
- 발견한 조합은 대표 카드를 바꾸거나 내려도 영구 보존한다. 잠긴 도감은 필요한 세 분야만 보여 주고
  이름과 기념 표식은 발견 전까지 숨긴다. 첫 발견·6종·18종은 이야기·수집·숙련 퀘스트로 연결한다.
- 현재 조합은 공개 마을 명함에만 함께 표시한다. 서버와 다른 이용자는 전문성 ID에서 조합을 다시
  계산하며 클라이언트가 보낸 임의 호칭을 신뢰하지 않는다.
- 별자리 카드는 180×96 저해상도 Canvas 렌더러를 사용한다. 데스크톱은 대표 덱과 설명을 2열로,
  모바일은 카드 3장·현재 시너지를 단일 열로 접어 가로 스크롤 없이 읽히게 한다.

### 1.2d-0 선택형 여정·마을 레벨 규격 (`src/game/progression/villageJourney.ts`)

- 전투 레벨과 별도로 모든 생활 숙련 XP와 완료한 영구 퀘스트를 합치는 **마을 Lv.1~50**을 둔다.
  집 꾸미기·코디·펫·관계·정원·요리·낚시만으로도 최고 레벨에 접근할 수 있어 전투를 강제하지 않는다.
- 여정은 8장으로 구성하되 각 장의 모든 목표를 요구하지 않는다. 4~10개 후보 중 3~5개만 골라
  완료하면 다음 장이 열린다. 이전 장이 잠겨 있을 때 먼저 수행한 기록도 나중에 소급 인정한다.
- 여정에는 실패·기한·진행 감소가 없다. 잠긴 장은 정확한 이전 장 이름을 보여 주고, 열린 목표는
  기존 HUD 추적과 연결해 한 번의 조작으로 다음 위치를 확인할 수 있어야 한다.
- 8개 장 완료와 마을 Lv.5·10·20·30·40·50은 장착형 배지 퀘스트로 연결한다. 이 보상 퀘스트는
  마을 XP 입력에서 제외해 자기 완료가 다시 레벨을 올리는 순환을 금지한다.
- 각 마을 레벨 명찰은 `villageRewards.ts`의 단일 카탈로그에서 코디 프리셋·펫 픽셀 장식·가구 리폼
  레시피를 한 묶음으로 자동 해금한다. 별도 수령·기한·재화 지급은 두지 않으며, 배지 ID를 공통
  권위값으로 사용해 옷장·펫샵·내 집 공방이 같은 잠금 상태를 보여야 한다.
- 데스크톱은 레벨·진행·다음 보상이 비대칭으로 보이는 여정 헤더를 사용하고, 모바일은 엄격한 단일 열로
  접는다. 모션 감소 설정에서는 장 순차 등장과 버튼 전환을 제거한다.

### 1.2d-0a 취향 프로필 규격 (`src/game/progression/tasteProfile.ts`)

- 가이드북 첫 화면은 패션·집 꾸미기·동행·정원·요리·낚시·이웃·탐험 여덟 분야의 실제 평생 기록을
  한 장에 집계한다. 임의 활동 점수나 추정치는 사용하지 않고 `QuestState.lifetimeCounts`만 읽는다.
- 각 분야는 2~3개의 유한 수집 목표와 정확한 사용 위치를 함께 보여 준다. 목표를 넘긴 원본 기록은
  보존하되 화면 퍼센트는 100에서 멈추며, 가장 진행된 분야를 현재 취향 호칭으로 사용한다.
- 새 이용자는 0%를 실패로 표현하지 않는다. 좋아하는 생활 하나를 시작하면 첫 취향이 기록된다는 빈
  상태와, 완성에 가장 가까운 다음 기록을 제공한다.
- 데스크톱은 프로필 인장·현재 취향·완성 통계를 비대칭 헤더로 두고 분야 기록은 2열로 나눈다. 700px
  이하에서는 탭을 가로 스크롤, 모든 기록을 단일 열로 접고 가로 넘침을 금지한다.
- 마을 레벨 보상 수첩의 코디·펫·가구 그림은 실제 게임 렌더러를 재사용한다. 잠긴 보상도 실루엣은
  보여 주되 회색 처리하고, 해금 뒤에는 같은 렌더 결과가 아틀리에·펫샵·리폼 공방에 나타나야 한다.

### 1.2d-0b 나의 수집 선반 규격 (`src/game/progression/collectionShelf.ts`)

- 이용자마다 최대 6개의 아이템·몬스터·취향 기록을 선반에 고정한다. 기한·실패·진행 감소는 없고,
  목표를 빼도 이미 얻은 아이템과 도감 기록은 지우지 않는다.
- 진행도는 별도 카운터를 만들지 않고 인벤토리 발견 기록·몬스터 연구 도감·평생 기록의 권위값을 읽는다.
  목표를 나중에 담아도 과거 행동을 즉시 소급 인정하며, 완료한 카드는 선반에 계속 남길 수 있다.
- 각 카드는 실제 획득 장소와 목표 수량을 함께 보여 준다. 가구는 `살림 가구`의 분류·가격을 공개하고,
  몬스터는 처음 발견하기 전까지 이름을 감춘 채 티어와 동쪽 외곽숲 해금 경로만 안내한다.
- 선반이 비었거나 6칸이 찬 상태를 인라인 문장으로 설명한다. 데스크톱은 비대칭 2열, 모바일은 엄격한
  단일 열을 사용하며 자동 등장은 transform·opacity만 사용하고 모션 감소 설정을 따른다.
- `길 안내`는 문구 링크가 아니라 현재 타일부터 실제 활동 지점까지 충돌을 피한 최단 금빛 발자국을
  월드에 그린다. 생활 물건은 살림 쇼룸 도착 시 해당 분류와 목표 진열대를 자동으로 펼친다.
- 구매 성공 뒤 서버 잔액·인벤토리·발견 도감·선반 완료를 한 흐름으로 갱신한다. 실패 시 코인과 물건이
  그대로임을 인라인으로 알려야 하며, 오프라인은 가격과 진열대를 볼 수 있는 둘러보기 모드를 유지한다.
- 쇼룸 방문·첫 구매·10/30/65종 도감·누적 50점 구매는 실패·기한 없는 6개 퀘스트로 이어지고, 과거
  구매와 발견 기록도 소급 인정한다.

### 1.2d-1 도시 정원 규격 (`src/game/garden/`, `gardenPanel.ts`)

- 소품 이미지 origin `(0.5, 1)` 바닥 중심, depth 2. 건물 파사드 depth 1, 문 depth 2.
- 아이소메트릭 가구는 `footprint(w×h)`·`anchor`·`elevation`·4방향 회전을 명시한다.
- **충돌 정합 필수**: "막혀 보이는 것은 막힌다." 큰 소품·기계·건물은 `SOLID_RECTS`/
  `SOLID_PROPS`에 등록. 문/입구는 collision hole로 뚫고 **보이는 문**을 그린다.
- 새 상호작용 스팟엔 통통 튀는 아이콘 + 라벨을 단다(어디서 뭘 하는지 명확히).

### 2.3 HUD
- 좌상단은 현재 퀘스트 길잡이, 우상단은 코인·시계·설정 기어, 하단은 나무 아이콘 바.
- 길잡이는 목표 이름만 보여 주지 않고 `추천 이유 → 다음 행동 → 위치 → 진행률` 순으로 읽힌다.
  자동 추천과 직접 추적을 라벨·테두리 색으로 구분하고, 누르면 모험 일지의 추천 탭으로 돌아간다.
- 아이콘은 **12×12 픽셀 아이콘**(`pixelIcons.ts`), 공통 외곽선 `#4a2c12`. 이모지 금지
  (아이콘 자리엔 픽셀 아이콘을 쓴다).
- 모바일: 하단 바 2단 그리드, 조이스틱(터치 지점에 출현).

### 2.3.1 모바일 조작·성능 편안함

- 첫 터치 진입에서는 `빈 곳을 누른 채 끌어 이동` 안내를 한 번 보여 주고, 실제 드래그나 확인 버튼으로
  닫는다. 고정 D-패드 대신 손가락을 놓은 위치에 가상 조이스틱을 띄운다.
- 방 꾸미기의 회전·선택 해제처럼 씬 고유 동작은 우하단 원형 버튼으로 제공하고 접근 가능한 이름을 붙인다.
- 그래픽 기본값은 `자동 맞춤`이다. 기기 힌트로 시작하되 실제 프레임 평균·95백분위·끊김 비율을 함께
  측정하고, 지속적인 저하에서만 `고품질 → 균형 → 발열 절약` 순으로 한 단계씩 조절한다.
- 어떤 단계에서도 네이티브 픽셀 해상도·좌표·충돌·진행 규칙은 낮추지 않는다. 절전은 반복 장식,
  그림자 농도, 장식성 전투 이펙트와 화면 투영 빈도만 줄인다.
- `prefers-reduced-motion`과 절전 단계에서는 첫 조작 안내의 등장 애니메이션을 생략한다.
- 설정의 `플레이 배려`는 화질과 별개로 큰 글씨 2단계, 고대비, 움직임 자동/전체/최소화를 제공한다.
  선택은 계정이나 보상과 분리된 `hv-play-comfort-v1` 기기 설정으로 저장해 로그인 전후에도 유지한다.
- 움직임 최소화는 CSS 애니메이션과 전환뿐 아니라 Phaser의 반복 월드 트윈, 전투 흔들림,
  레벨업·사진 플래시와 장식성 전투 이펙트를 함께 끈다. 정보성 토스트·피격 색·진행 수치는 유지한다.
- 접근성 선택을 퀘스트·배지·능력치·랭킹 조건으로 사용하지 않는다. 어떤 조합도 플레이 손해 없이
  즉시 바꿀 수 있고, OS의 모션 감소 변경은 `자동` 상태에서 즉시 다시 적용한다.

### 2.4 텍스트 (가독성 필수)
- 폰트: **`UI_FONT`**(한글 고딕 스택, `config.ts`). Phaser 기본 Courier 금지.
- 게임 내 텍스트는 `resolution: TEXT_RES`(DPR) 지정 — 고해상도에서 뭉개짐 방지.
- 최소 크기: 게임 내 8px+, 패널 본문 12px+. 7px 이하 금지.
- 말풍선: 크림 라운드 + 꼬리, 진갈 텍스트, 뿅 등장.

### 2.5 네컷 작업실
- 사진은 120×268 논리 픽셀 캔버스에 4개 장면을 합성하고 CSS에서 픽셀 보간 없이 확대한다.
  캐릭터는 `paintCharacterFrame`, 펫은 `paintPet`을 사용해 월드·아틀리에와 동일한 레이어를 공유한다.
- 배경은 골목·작업실·정원·방·무대·야간 6종이며 프레임은 오트밀·숲길·살구·호두·잉크·금박 6종이다.
  프레임은 앨범 사진 수로 순차 해금하고 잠긴 이유와 필요한 장수를 항상 함께 표시한다.
- 저장 레코드는 외형·펫·프레임·배경·포즈의 당시 스냅샷이다. 이후 코디나 펫 장식을 바꿔도 과거 사진을
  덮어쓰지 않는다. 12칸이 차면 자동 삭제하지 않고 명시적으로 정리하도록 안내한다.
- 데스크톱은 미리보기와 편집기의 비대칭 2열, 760px 이하는 단일열이다. 삭제는 2회 확인,
  빈 앨범은 촬영 탭으로 돌아가는 명확한 복구 행동을 제공한다.

### 2.6 골목 룩북
- 아틀리에의 `룩북` 탭은 12개 코디 의뢰를 왼쪽 색인과 오른쪽 종이 의뢰서의 비대칭 구조로 표시한다.
  각 의뢰는 정확히 세 조건만 사용하며 현재 코디의 `일치/조정`을 색·마름모 표식·텍스트로 함께 전달한다.
- 첫 네 장은 즉시 공개하고 이후 의뢰는 기존 생활 배지와 연결한다. 잠긴 카드에는 필요한 배지 이름을
  노출하며, 샘플 코디는 조건에 맞는 값만 미리 입히고 저장이나 제출을 자동으로 수행하지 않는다.
- 제출은 실패하지 않는다. 조건을 하나도 맞추지 못해도 참여 별 하나를 기록하고, 낮은 재도전이 기존
  최고 별을 덮어쓰지 않는다. 최고 기록 프레임은 당시 외형 스냅샷을 `paintCharacterFrame`으로 재현한다.
- 의뢰별 `ink/paper` 두 색으로 종이 재질을 바꾸되 한 화면의 강조색은 해당 잉크 하나만 사용한다.
  별은 이모지 대신 CSS 마름모 세 개로 그린다. 700px 이하는 단일열, 430px 이하는 기록 프레임과
  조건 목록도 단일열로 접어 문서 전체의 가로 스크롤을 만들지 않는다.

### 2.7 골목 의뢰소
- 이웃·골목·동행·스타일·나의 집·손작업·물정원·모험 8분야에 정확히 3장씩, 총 24개 수제 의뢰를 둔다.
  기존 평생 퀘스트 지표의 **수락 이후 증가분**만 읽어 과거 플레이가 새 의뢰를 즉시 완료하지 않게 한다.
- 활성 슬롯은 세 장이다. 진행 중 의뢰를 잠시 보관하면 그 시점의 진행을 `carry`로 저장하고, 다시
  이어갈 때 새 기준점을 사용한다. 기간 만료·실패·진행 감소·재화 페널티는 없다.
- 첫 완료는 24칸 고유 도장을 열고 재완료는 누적 해결과 평판을 올린다. 분야 도감과 평판 Lv.1~10은
  모험 일지 장기 퀘스트 및 이웃 관계 생활 숙련에 연결한다.
- 화면은 600×210 절차형 픽셀 게시판, 현재 의뢰 3열, 24장 종이 도감의 비대칭 구조다. 700px 이하에서는
  한 열로 접고 8px 미만 텍스트·이모지 아이콘·가로 스크롤을 사용하지 않는다.

### 2.8 골목 가구 리폼 공방
- 방에 실제 배치된 가구·가전·소품·러그·벽장식만 작업 목록에 올린다. 식물과 생물이 사는 어항은
  원본 외형을 보존한다. 위치·회전·소유권·홈 점수는 바꾸지 않고 placement ID에 코스메틱 외형만 연결한다.
- 재질 마감 6종과 색감 8종의 48조합을 제공한다. 잠긴 선택지는 숨기지 않고 홈·낚시·정원·요리·보물 등
  기존 생활 기록에서 필요한 조건과 현재값을 함께 표시한다. 리폼에는 코인·소모 재료·실패가 없다.
- 실제 방 텍스처는 원본 가구 픽셀 위에 색 혼합과 마감 패턴을 합성한다. 미리보기와 방 렌더가 같은
  `FurnitureReformStyle`을 사용하며, 선택한 조합은 새 도감 기록 여부와 관계없이 즉시 다시 적용할 수 있다.
- 데스크톱은 작업대와 재료 서랍의 비대칭 2열, 860px 이하는 단일열이다. 모든 텍스트는 8px 이상이며
  상태는 색만으로 전달하지 않고 이름·설명·잠금 조건을 함께 쓴다. 이모지 아이콘은 사용하지 않는다.

### 2.9 2.5D 실내 디오라마
- 실내도 거리와 같은 64×32 다이아몬드를 사용한다. 저장·충돌·서버 RPC는 기존 32px 논리 좌표를
  유지하고 `isometricRoom.ts` 경계에서만 투영·역투영해 기존 방과 홈 장면 보관함을 무손실 보존한다.
- 배치 회전은 북·동·남·서 4방향이다. 기존 `rot=0/1`은 북·동으로 그대로 해석하고 2/3만 확장해
  과거 방을 다시 저장하지 않아도 한다. 벽장식은 벽면 방향 고정이라고 고스트에 명시한다.
- 북·서 벽만 남긴 오픈월 인형집 구조로 방 전체를 읽게 한다. 내부 칸막이는 타일별 입체 프리즘으로
  분리하고 캐릭터·펫·가구와 같은 화면 y depth 공식을 써 앞뒤를 걸을 때 자연스럽게 가려진다.
- 가구 텍스처는 점유 다이아몬드 중앙 남쪽 기준선에 고정한다. 러그는 직사각 이미지를 늘이지 않고
  풋프린트 다이아몬드로 다시 합성하며, 회전 방향에 따라 내부 직조선도 함께 바뀐다.
- 바닥 가구만 이동 충돌을 만들고 러그·벽장식은 통과 가능하다. 오래된 저장 때문에 입장 지점이 가구와
  겹치면 가장 가까운 안전 타일로 조용히 이동해 갇힘을 방지한다. 플레이어가 서 있는 칸에는 새 가구를 놓지 않는다.
- 입력은 화면 방향을 논리 격자 방향으로 회전하고 포인터는 카메라 줌·스크롤을 보정해 역투영한다.
  가구 고스트는 정확한 다이아몬드 점유 영역과 방향, 배치 가능 여부를 색과 텍스트로 함께 표시한다.

### 2.10 살림 쇼룸 획득망
- 65종은 기본 진열 37종, 주간 큐레이션 16종, DIY 12종으로 중복 없이 나눈다. 주간 큐레이션은
  KST 월요일 00:00에 4점씩 교체되고 4주 안에 모두 다시 등장해 놓침에 대한 불안을 만들지 않는다.
- 주간 수첩은 현재 4점뿐 아니라 다음 1~3주의 12점과 재등장 주차를 모두 보여 준다. 구매 이력은
  거래 원장에서 복원해 판매해도 수집 완료가 되돌아가지 않는다.
- DIY 카드는 완성품, 재료별 보유/필요 수량, 공임, 부족 원인을 함께 보여 준다. 재료 차감·공임 결제·
  완성품 지급·제작 이력 기록은 advisory lock을 건 서버 RPC 한 번으로 처리한다.
- 데스크톱은 비대칭 2열 작업일지, 700px 이하는 엄격한 단일열이다. 신규 UI에 이모지 아이콘을 쓰지 않고,
  등장 모션은 transform·opacity로만 만들며 reduced-motion에서는 제거한다.
- 아틀리에 앞 `diy_workbench_v1.png`는 128×128 RGBA·2:1 아이소메트릭 생성형 픽셀 에셋이다. 로드에
  실패하면 동일 footprint의 절차형 원목 작업대로 폴백하며, 전용 활동 타일은 쇼룸 구매 타일과 분리한다.
- 레시피 완성품과 재료는 색상칩이 아니라 `ensureFurniture`가 만드는 실제 방 텍스처를 DOM용 투명 PNG로
  구워 표시한다. 재료마다 획득 진열대로 이동하는 명시적 버튼을 제공하고 색만으로 부족 상태를 전하지 않는다.

### 2.11 골목 나눔장터
- 가격은 카탈로그 정가의 40% `이웃가`, 60% `포근가`, 80% `소장가` 세 단계뿐이다. 사용자가 임의 가격·
  자유문구·흥정 조건을 입력하는 UI를 만들지 않으며 서버가 최종 가격을 다시 계산한다.
- 등록 시 인벤토리 한 점을 서버 에스크로로 이동하고, 판매 취소 시 즉시 돌려준다. 구매자 코인 차감·판매자
  정산·아이템 이전·거래 원장은 잠금이 걸린 단일 서버 트랜잭션으로 처리한다. 직접 테이블 쓰기는 허용하지 않는다.
- 한 사람의 활성 판매는 6점, 개인 찜은 8점이다. 찜은 예약·선점이 아니며 순위·별점·기간 한정·거래 실패
  페널티·취소 수수료를 두지 않는다. 자기 판매품 구매와 이미 정산된 카드의 중복 구매는 서버에서 거절한다.
- 이웃 선반의 카드는 실제 가구 픽셀, 분류, 고정 가격 단계, 판매자, 정가를 함께 표시한다. 구매와 등록은
  반드시 2단계 확인을 거치며 처리 중 중복 입력을 막는다. 색만으로 상태를 전달하지 않는다.
- 오프라인은 6점 샘플 선반과 로컬 찜 수첩만 제공하는 읽기 전용 미리보기다. 실제 코인·인벤토리·거래 기록은
  변경하지 않는다. 방문·찜·등록·구매·판매·분류·물건·교환은 영구 퀘스트와 이웃 관계 숙련에 연결한다.
- 상단 아트는 280×112 절차형 픽셀 야시장으로 실제 진열과 찜 색을 반영한다. 데스크톱은 비대칭 선반 그리드,
  680px 이하는 단일열이며 이모지 대신 한글 도장과 실제 아이템 미리보기를 사용한다.

## 5. AI 이미지 생성 워크플로 (소품·에셋)

### 5.1 사용 도구 (허용)

### 5.2 금지

### 5.3 생성 프롬프트 규격 (일관 스타일 고정)
소품/건물 생성 시 프롬프트에 항상 포함:
- `cozy detailed isometric pixel art game asset, strict 2:1 projection, 3/4 view`
- `warm muted palette (soft browns, cream, sage), low saturation, chunky clean pixels`
- `single object centered on PURE WHITE background, no external ground shadow, no text`
- 캐릭터·네온·사진풍·3D 렌더 금지 명시.
- 크기: 타일 배수로 요청(1×1, 2×2, 3×2 등)해 맵 타일에 맞춘다.

### 5.4 후처리 파이프라인 (`tools/pixelize.py`)
생성 이미지는 반드시 픽셀화·규격화 후 적용:
1. center-crop → LANCZOS 2x → NEAREST → median-cut 양자화(팔레트 축소).
2. `--white-key`로 흰 배경 → 투명 처리 + bbox + 바닥 정렬.
3. `public/assets/...`에 저장 → `assetManifest`에 등록(프로시저럴 폴백 유지).

## 6. 코드·구현 규격

- **순수 로직 분리 + 테스트**: 게임 규칙(경제·충돌·평면·근무·보물)은 순수 함수로 빼고

- **서버 권위**: 코인·소유·계약 등 경제는 security-definer RPC로만. 클라 직접 쓰기 금지.
  클라는 RPC-우선 + 미적용/오프라인 우아한 폴백.
- **지속 상태**: 진행·수집은 localStorage 스토어(userId별, KST 자정 리셋 규칙 공유).
- **퀘스트 추적**: 유효한 `focusedQuestId`만 userId별로 보존한다. 잠긴 목표는 선택할 수 없고,
  직접 추적한 목표가 완료되면 자동으로 다양한 추천 흐름으로 복귀한다. 일일 목표 추적은 자정 뒤에도 유지된다.
- **사진 앨범**: 완성 PNG 대신 재현 가능한 외형·스타일 스냅샷만 userId별로 보관한다. 손상 레코드,
  알 수 없는 펫·프레임·포즈는 정규화 단계에서 제거하거나 안전한 기본값으로 복구한다.
- **평생 기록과 표시 분리**: 영구 행동 횟수는 목표 이후에도 보존하고, 개별 퀘스트 UI에서만 `goal`로 제한한다.
- **작업 흐름**: main에서 새 브랜치 → 구현 → typecheck+vitest → Playwright E2E(데스크톱·
  모바일) → PR → CI → 스쿼시 머지 → 자동 배포 → 실서비스 200 확인. 커밋은 요청 시에만
  하되 이 프로젝트는 매 기능마다 위 파이프라인으로 배포까지가 "완료".
- 실서버 경제 변경 시 마이그레이션 SQL을 사용자에게 전달(사용자가 실행).
