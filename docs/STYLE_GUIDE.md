# 홍대마을 — 톤앤매너 규격 (STYLE GUIDE / SSOT)

> 이 문서는 홍대마을의 **단일 진실 원천(SSOT)** 이다. 모든 신규 아트·UI·카피·사운드·
> AI 생성물은 여기 정의된 규격을 **반드시** 따른다. 규격을 바꿔야 하면 이 문서를 먼저
> 고치고, 코드/에셋을 맞춘다. "항상 같은 스타일"이 목표다.
>
> 우선순위: 사용자 지시 > **이 문서** > 개별 파일 관행 > 기본값.

---

## 0. 한 줄 컨셉

**서울 홍대입구, 차분하고 아기자기한 웜톤 픽셀 코지 마을.** 주인공 건물은 **마인드
포레스트(인싸이트) 빌딩**. 동물의 숲·스타듀밸리 감성의 힐링·수집·소장.

---

## 1. 아트 원칙

- **픽셀아트 only.** 벡터·사진·3D 렌더 금지. `pixelArt: true`, `image-rendering: pixelated`.
- **무드**: 차분한 웜톤(따뜻한 갈색·크림·세이지). 채도 낮게, 대비 부드럽게. 네온·형광 금지.
- **프로시저럴 우선 + AI 보강**: 모든 아트는 코드 프로시저럴로 폴백이 존재해야 한다.
  AI 스프라이트는 있으면 얹고, 없어도 게임이 동일하게 돌아가야 한다(404 안전).

### 1.1 게임 아트 팔레트 (SSOT = `src/game/art/palette.ts`)
프로시저럴 지형·건물·캐릭터·방은 **`PAL` / `ROOM_PAL` 안에서만** 색을 고른다. 새 색이
필요하면 팔레트에 먼저 추가한다. 대표값:
- 바닥: `alley 0x8a7d6e`, `grass 0x7d9468`, `plaza 0x9a8d7c`, `road 0x5c5852`
- 건물: `wall 0xc4a98a~0x8fa38a`, `roof 0x6e5c4c`, `door 0x8a5a3a`, `winGlassWarm 0xf2d8a0`
- 캐릭터: `skin 0xf0cfae`, `outline 0x3a2f26`
- 방: `floorWood 0xa8896a`, `wallPaper 0xd8cbb6`

### 1.2 캐릭터 규격 (`src/game/art/characterArt.ts`)
- 24×32px, 4방향 × 4프레임 시트, 걷기 애니 `[1,2,3,2]`.
- 외형 조합: 피부6 × 헤어6 × 헤어색12 × 상의24 × 하의12 × 안경5 × 모자8 (`appearance.ts`).
  실루엣은 통일, 색·소품만 변주. 아치형·비율 왜곡 금지.
- 레이어 순서: 다리→몸통→팔→머리→헤어→**안경**(눈 위, 뒤통수 제외)→**모자**(정수리).
  안경·모자는 옵션(0=없음)이며 레거시 저장값엔 없어도 0으로 정규화된다.
- 커스터마이즈 패널은 **큰 픽셀 미리보기 필수**(`paintCharacterFrame`)로 방향 회전·즉시 반영.
- origin `(0.5, 0.66)` — 발이 충돌 박스 바닥에 오게. depth: 플레이어 10, NPC 9.

### 1.2b 펫 규격 (`src/game/art/petArt.ts` / `pets/`)
- 16×16px, **우향 1프레임**(좌향은 스프라이트 flipX). 종별 귀(`earType`)·꼬리(`tail`)·색
  파라미터로 실루엣 구분. 캐릭터와 동일한 아웃라인·볼터치 감성 유지.
- 팔로워는 위치 이력 버퍼로 몇 프레임 뒤처져 따라오고 멈추면 바운스(동물의 숲 감성).
  depth 9(플레이어 아래). 동행 상태·보유는 `PetStore`(로컬) + 서버 `owned_pets`(0011) 병합.
- 가격 SSOT = 서버 `adopt_pet` RPC. `pets.ts` 가격과 SQL을 **항상 동일하게** 유지.

### 1.3 소품·건물 배치 규격
- 타일 = 32px(`TILE`). 좌표·크기는 타일 단위로 정의(`mapData.ts`).
- 소품 이미지 origin `(0.5, 1)` 바닥 중심, depth 2. 건물 파사드 depth 1, 문 depth 2.
- **충돌 정합 필수**: "막혀 보이는 것은 막힌다." 큰 소품·기계·건물은 `SOLID_RECTS`/
  `SOLID_PROPS`에 등록. 문/입구는 collision hole로 뚫고 **보이는 문**을 그린다.
- 새 상호작용 스팟엔 통통 튀는 아이콘 + 라벨을 단다(어디서 뭘 하는지 명확히).

### 1.4 젬·희귀도 (`treasures.ts` / `treasureIcons.ts`)
- 희귀도 4단계 색: 일반 `#8ca0b0` · 희귀 `#5aa668` · 영웅 `#a86ad0` · 전설 `#e8a83c`.
- 보물 아이콘은 `drawGem()` 파세트 젬 렌더로 통일(도감·HUD·방 진열장 공용). 새 보물도
  같은 렌더러를 쓴다.

## 2. UI 규격 — "나무 프레임" 시스템

모든 패널은 **따뜻한 나무 프레임** 톤으로 통일한다. 신규 UI는 CSS 변수(`overlay.css`
`:root`)와 아래 공통 클래스를 재사용한다.

### 2.1 UI 컬러 토큰 (CSS `:root` 변수 = SSOT)
| 토큰 | 값 | 용도 |
|---|---|---|
| `--wood-lite` | `#e7bd7f` | 버튼·칩 밝은 나무 |
| `--wood` | `#d9a45b` | 나무 기본 |
| `--wood-dark` | `#5c3a1c` | 나무 그림자·테두리 |
| `--wood-edge` | `#6e4522` | 버튼 테두리 |
| `--frame` | `#b07a46→#96613a` | 패널 프레임 그라디언트 |
| `--paper` | `#f2dcae` | 패널 내지(밝은 종이) |
| `--paper-2` | `#e0c088` | 슬롯·카드 바탕 |
| `--ink` | `#4a2e14` | 본문 텍스트(진갈) |
| `--cream` | `#fff2d8` | 헤더·라벨 밝은 글자 |
| `--accent` | `#e8a83c` | 강조(금빛) |
| `--gold-btn` | `#f2c25c→#e8a83c` | 확인·닫기 골드 버튼 |
| 회사 UI | `#3a4a6a`계열 파랑 | CompanyScene 전용(오피스 톤) |

### 2.2 공통 컴포넌트 (재사용 클래스)
- `.hv-wood-modal` + `.wood-frame`: 표준 나무 팝업. `.wood-head`(h2 크림+그림자, `.pill` 칩),
  `.wood-page`(종이 내지, 스크롤), `.wood-tabs`(탭), `.wood-close`(골드 ✕).
- 버튼: 나무 그라디언트 + 2~3px 테두리 + `box-shadow: 0 3px 0 그림자`. `:active`는
  `translateY(2px)` + 그림자 축소(눌리는 느낌).
- 라운드: 패널 14~16px, 카드/버튼 8~12px.
- **등장 애니**: 패널은 `hv-pop`(scale 0.85→1, back-ease), 말풍선은 `Back.easeOut` 뿅.

### 2.3 HUD
- 좌상단 하트(퀘스트 진행), 우상단 코인·시계·설정 기어, 하단 나무 아이콘 바.
- 아이콘은 **12×12 픽셀 아이콘**(`pixelIcons.ts`), 공통 외곽선 `#4a2c12`. 이모지 금지
  (아이콘 자리엔 픽셀 아이콘을 쓴다).
- 모바일: 하단 바 2단 그리드, 조이스틱(터치 지점에 출현).

### 2.4 텍스트 (가독성 필수)
- 폰트: **`UI_FONT`**(한글 고딕 스택, `config.ts`). Phaser 기본 Courier 금지.
- 게임 내 텍스트는 `resolution: TEXT_RES`(DPR) 지정 — 고해상도에서 뭉개짐 방지.
- 최소 크기: 게임 내 8px+, 패널 본문 12px+. 7px 이하 금지.
- 말풍선: 크림 라운드 + 꼬리, 진갈 텍스트, 뿅 등장.

## 3. 카피·보이스 (글쓰기 톤)

- 언어: **한국어**. 따뜻하고 아기자기하게. 존댓말 기본, 상황 따라 반말·구어체 OK.
- NPC 대사는 **세대별 말투**(10대 급식체·MZ 신조어·성인·시니어) + **실시간 시간대** 반영
  (`npcChatter.ts`). 트렌디하게, 과하지 않게.
- 이모지는 **양념으로만** — 문장당 0~1개. 남발 금지.
- 실제 홍대/브랜드/회사(인싸이트·학지사·학지사 에듀·AX기획실 팀원)는 실제대로 반영.
- 유머는 사내 감성·홍대 감성으로 소소하게(예: "전 직원 하와이 워크샵 → 반려 ㅋㅋ").

## 4. 사운드 규격 (`music.ts` / `audio.ts`)

- BGM: **로파이 코지** 루프(72BPM, Cmaj7–Am7–Fmaj7–G7, 핑거피킹+패드+차임). 잔잔·힐링.
- 효과음: WebAudio 합성(외부 파일 X). 부드러운 blip. 코인·성공·문·음표.
- 볼륨/진동은 설정 슬라이더 + localStorage. iOS는 제스처마다 resume.

## 5. AI 이미지 생성 워크플로 (소품·에셋)

### 5.1 사용 도구 (허용)
- **미드저니(Midjourney)** · **제미나이(Gemini)** · **챗지피티(ChatGPT/DALL·E)**.
- 위 3종으로 생성한 이미지를 아래 파이프라인으로 규격화해 적용한다.

### 5.2 금지
- **higgsfield 사용 금지** — 사용자가 다시 허용하기 전까지 쓰지 않는다.

### 5.3 생성 프롬프트 규격 (일관 스타일 고정)
소품/건물 생성 시 프롬프트에 항상 포함:
- `cozy pixel art game asset, top-down / 3-4 view (Stardew-like)`
- `warm muted palette (soft browns, cream, sage), low saturation, chunky clean pixels`
- `single object centered on PURE WHITE background, no external ground shadow, no text`
- 캐릭터·네온·사진풍·3D 렌더 금지 명시.
- 크기: 타일 배수로 요청(1×1, 2×2, 3×2 등)해 맵 타일에 맞춘다.

### 5.4 후처리 파이프라인 (`tools/pixelize.py`)
생성 이미지는 반드시 픽셀화·규격화 후 적용:
1. center-crop → LANCZOS 2x → NEAREST → median-cut 양자화(팔레트 축소).
2. `--white-key`로 흰 배경 → 투명 처리 + bbox + 바닥 정렬.
3. `public/assets/...`에 저장 → `assetManifest`에 등록(프로시저럴 폴백 유지).

## 6. 코드·구현 규격

- **순수 로직 분리 + 테스트**: 게임 규칙(경제·충돌·평면·근무·보물)은 순수 함수로 빼고
  vitest로 요구사항을 못박는다. 씬은 얇게(렌더·입력·배선만).
- **서버 권위**: 코인·소유·계약 등 경제는 security-definer RPC로만. 클라 직접 쓰기 금지.
  클라는 RPC-우선 + 미적용/오프라인 우아한 폴백.
- **지속 상태**: 진행·수집은 localStorage 스토어(userId별, KST 자정 리셋 규칙 공유).
- **작업 흐름**: main에서 새 브랜치 → 구현 → typecheck+vitest → Playwright E2E(데스크톱·
  모바일) → PR → CI → 스쿼시 머지 → 자동 배포 → 실서비스 200 확인. 커밋은 요청 시에만
  하되 이 프로젝트는 매 기능마다 위 파이프라인으로 배포까지가 "완료".
- 실서버 경제 변경 시 마이그레이션 SQL을 사용자에게 전달(사용자가 실행).

## 7. 체크리스트 (신규 작업 전 확인)

- [ ] 색은 팔레트/CSS 토큰에서 골랐다(임의 hex 지양).
- [ ] UI는 나무 프레임 공통 클래스를 재사용했다.
- [ ] 텍스트에 `UI_FONT` + `resolution` + 최소 크기.
- [ ] 픽셀 아이콘(이모지 아님) / 젬은 `drawGem`.
- [ ] "막혀 보이면 막힘" 충돌 정합 + 보이는 문/스팟 라벨.
- [ ] 카피는 한국어 웜톤, 이모지 절제, 세대·실시간 반영.
- [ ] AI 생성은 미드저니/제미나이/챗지피티 + pixelize 파이프라인. **higgsfield 금지.**
- [ ] 순수 로직 테스트 + 프로시저럴 폴백 + E2E + 배포 확인.
