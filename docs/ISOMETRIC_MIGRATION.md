# 홍대마을 아이소메트릭 전환 로드맵

## 목표

기존 게임의 로그인·경제·레벨·전투·퀘스트·멀티플레이를 유지하면서 화면과 배치 구조를
고밀도 2.5D 아이소메트릭 픽셀아트로 단계 전환한다. 참고 작품은 품질 기준으로만 사용하며,
홍대 골목과 한국 생활 공간을 중심으로 독자적인 디자인을 만든다.

## 변하지 않는 계약

- 서버와 Realtime은 기존 32px 논리 월드 좌표를 사용한다.
- 직교 `street`와 `iso-village`는 좌표 범위가 달라 Realtime presence 채널을 분리한다.
- 충돌과 이동은 32px 직교 논리 격자를 사용한다.
- 코인·소유권·레벨·경험치·퀘스트 데이터는 그대로 유지한다.
- DOM 패널 UI는 씬 전환과 무관하게 재사용한다.

## 새 렌더링 계약

- 화면 타일: 64×32px, 2:1 다이아몬드
- 화면 투영: `projectIso` / `projectIsoWorld`
- 클릭 역투영: `unprojectIso` / `pickIsoTile`
- 가림 순서: 풋프린트의 남쪽 모서리 기반 `isoDepth`
- 높이: 16px 단위 elevation
- 화면 기준 입력: `screenInputToWorld`

## 병합 전략

아이소메트릭 기반은 기존 기능 파일을 최소 수정하고 새 모듈로 추가한다. `main`의 후속 기능을
주기적으로 브랜치에 병합해 충돌을 작게 유지하며, 수직 슬라이스가 승인되기 전에는 기본 시작
경로를 변경하지 않는다. 최종 전환은 씬 단위 PR로 나눠 병합한다.

마을 레벨 보상은 DB 지급 레코드를 추가하지 않고 기존 장착형 배지 ID를 해금 권위값으로 재사용한다.
따라서 `villageRewards.ts`와 세 소비 화면(옷장·펫샵·리폼 공방)을 같은 커밋 단위로 옮기면 되며,
원격 스키마 마이그레이션 없이 메인 작업과 취합할 수 있다.

## 성능 편안함 계약

- `performance/performanceComfort.ts`가 실제 Phaser `delta`의 평균·95백분위·25ms 초과 비율을
  이동 창으로 측정한다. 탭 복귀·디버거 정지처럼 100ms를 넘는 표본은 판단에서 제외한다.
- 자동 모드는 지속적인 저하가 두 번 관찰될 때만 한 단계 낮추고, 회복은 여섯 번 확인한 뒤 올린다.
  짧은 로딩이나 이펙트 한 번으로 품질이 출렁이지 않는다.
- `high / balanced / battery` 모두 논리 좌표·충돌·픽셀 해상도·게임 규칙은 동일하다.
  낮은 단계는 반복 장식 속도, 그림자 농도, 전투 오브젝트 화면 투영 빈도와 장식 이펙트만 줄인다.
- 설정의 `자동 맞춤 / 고품질 / 균형 / 발열 절약` 선택은 로컬에 보존되며, 실측 FPS·끊김 비율과
  적용 이유를 함께 보여 준다.

## 플레이 배려 계약

- `accessibility/playComfort.ts`가 글씨 크기·색상 대비·화면 움직임을 기기 단위로 보존하고,
  `data-hv-text / data-hv-contrast / data-hv-motion`을 모든 DOM 패널의 공통 적용점으로 사용한다.
- 모션 `자동`은 `prefers-reduced-motion`을 따르며, 직접 선택한 `부드럽게 / 최소화`가 OS 값보다
  우선한다. OS 설정이 실행 중 바뀌어도 자동 모드에서는 즉시 월드와 UI를 함께 갱신한다.
- 아이소메트릭 월드는 모션 최소화에서 주민·활동 표식 반복 트윈과 전투 장식 이펙트를 멈추고,
  피격 흔들림·레벨업 플래시·사진 플래시를 만들지 않는다. 레거시 월드도 전투 흔들림과 플래시를
  같은 전역 설정으로 차단한다.
- 배려 설정은 데이터 초기화 대상이 아니며 퀘스트, 배지, 레벨, 경제, 전투 판정과 완전히 분리한다.
