# 홍대마을 (Hongdae Village) — 시즌1 디자인 스펙

- 작성일: 2026-07-08
- 상태: 사용자 승인 대기
- 가제: 「홍대마을」 (정식 타이틀은 구현 중 별도 결정, 코드/저장소명은 `hongdae-village`)

## 1. 개요

최대 10인이 함께 지내는 웹 브라우저 기반 픽셀아트 힐링 멀티플레이 게임.
서울 홍대입구역 일대를 배경으로, 각자의 공간을 꾸미고 아이템을 구매·제작·거래하며
작은 사회를 만들어가는 잔잔한 생활 게임. 참조 감성: 〈위스퍼 오브 더 하우스〉.

### 확정된 핵심 결정

| 항목 | 결정 |
|---|---|
| 플랫폼 | 웹 브라우저 (데스크톱 우선) |
| 엔진/스택 | Phaser 3 + TypeScript + Vite |
| 백엔드 | Supabase (Auth·Postgres·Realtime·Edge Functions) 단일 스택, 별도 게임서버 없음 |
| 아트 | 고밀도 커스텀 픽셀아트 (위스퍼 결), AI 초안 + 자동 정제 + 터치업 파이프라인 |
| 시점 | 거의 수직의 스트레이트 탑다운 (가구 전면 살짝 보임) |
| 공간 구조 | 공유 거리(홍대) + 개인 방 10채 |
| 동시접속 | 최대 10인 |
| 경제 | 잔잔한 활동형 (알바·버스킹·의뢰·일일 퀘스트) |
| 로그인 | 카카오 OAuth (구글 보조) |
| 소통 | 말풍선 채팅 + 이모트 8종 (근접 기반) |
| 조작 | WASD 이동 + 마우스 클릭 (상호작용·배치) |
| 배포 | GitHub 저장소 + GitHub Actions CI + Vercel 정적 배포 |

## 2. 월드 — 시즌1: 홍대입구역 일대

약 80×60타일(도보 왕복 2~3분) 규모. 실제 지리의 고증보다 "홍대의 감성 요소"를 압축.

| 구역 | 내용 |
|---|---|
| 홍대입구역 9번 출구 | 스폰 지점, 마을 게시판(공지·방명록·의뢰), 우편함, 만남의 광장 |
| 메인 스트리트 (걷고싶은거리) | 버스킹 스팟, 가구점 「살림」, 카페 「모퉁이」(알바), 편의점(24h), 소품샵 「잡화점」(주간 한정 로테이션) |
| 주택 골목 (동·서) | 개인 공간 10채 — 옥탑방·반지하·다세대·오피스텔·상가주택 2층 등 서로 다른 구조. 입주는 선착순 취향 선택 |
| 경의선 숲길 | 힐링 존 — 벤치·피크닉·계절 나무·산책 미션 |
| 포차 골목 | 밤(게임 내 실제 밤 시간)에만 열림. 야식·건배·모임 스팟 |
| 벽화 골목 | 플레이어 공동 벽화 (주 단위 테마), 사진 스팟 |

### 살아있는 거리

- **실제 시간 연동**: Asia/Seoul 기준 새벽/아침/낮/노을/밤 5단계. 조명·팔레트 틴트 변화. 포차 골목 밤 오픈, 편의점 24시간.
- **실제 날씨 연동**: OpenWeatherMap 서울 마포구 실황 (Edge Function 프록시, 10분 캐시). 맑음/흐림/비/눈 4상태 — 비 오면 NPC 우산, 창문 빗방울, 빗소리 앰비언트.
- **NPC 앰비언트**: 시간대별 행인(출근길 직장인→관광객→밤의 버스커·연인), 길고양이, 비둘기, 배달 오토바이. 배경의 숨결 역할(깊은 상호작용 없음).

## 3. 코어 루프 & 시스템

### 하루의 리듬 (세션 20~40분 기준)

접속 → 우편·게시판 확인 → 의뢰 2~3개 수행 → 상점 구경 → 내 방 꾸미기 →
친구 방 방문/거리 마주침 → 숲길·포차에서 머물다 로그아웃.

### 수익 활동 (잔잔한 활동형)

- **카페 알바 미니게임**: 주문 순서대로 재료 조합하는 잔잔한 리듬. 하루 3회 제한(과몰입 방지).
- **버스킹**: 스팟에서 간단한 리듬 입력. NPC·친구가 코인 투척, 친구 관람 시 보너스.
- **이웃 의뢰**: NPC(또는 친구)가 게시판에 쪽지 의뢰 — 화분 물주기, 택배 전달, 방 정리 돕기.
- **일일 퀘스트**: 도토리 줍기, 친구와 건배하기 등 소소한 목표.

### 함께하는 미션 (힐링 톤 유지)

- **협동 벽화**: 큰 벽을 픽셀 단위로 함께 채움. 주 단위 테마.
- **공동 꾸미기 의뢰**: 마을 공동 프로젝트(예: 주말까지 숲길에 벤치 3개+꽃길). 참여자 전원 보상.
- **집들이**: 방 완성 후 개최 → 방문 친구가 방명록·선물 남김.

### 꾸미기 & 아이템

- 격자 기반 자유 배치. 레이어: 바닥/벽지/가구/소품/벽걸이/조명. 회전·겹침 규칙·되돌리기.
- 획득 경로: ① 상점 구매 ② 잡화점 주간 한정 로테이션 ③ 크래프팅(재료 수집→DIY 가구).
- **카탈로그는 데이터 주도**: `items/*.json`(이름·카테고리·가격·획득처·배치 규칙) + 스프라이트만 추가하면 코드 수정 없이 게임 반영. 시즌 업데이트의 기반.

### 거래

- **1:1 직거래**: 마주 본 상태에서 거래창. 아이템↔아이템/코인. Edge Function 검증.
- **중고 장터**: 게시판 등록형(당근마켓 감성). 오프라인 중에도 거래 성사.

### 리텐션 장치

- 방명록·우편(오프라인 친구에게 선물+쪽지 → 다음 접속의 설렘)
- 캐릭터 커스터마이징 (베이스 바디 + 헤어/상의/하의/신발/소품 레이어, 시즌1 조합 수백 종)
- 사진 모드 (필름 프레임 스크린샷)
- 단골 시스템 (자주 가는 장소의 NPC가 반응 변화 — "늘 마시던 걸로?")
- 계절 이벤트 (실제 절기 연동: 벚꽃·장마·첫눈)

### 소통

말풍선 채팅 + 이모트 휠 8종(하트·웃음·박수·춤·놀람·졸림·건배·손흔들기). 근접 기반(같은 화면).

## 4. 아트 디렉션 & 파이프라인

### 스타일 가이드 (전 에셋 공통)

- 기준 타일 **32×32px**, 화면 2배 확대 렌더링
- 시점: 거의 수직 탑다운 + 가구 전면 살짝
- **마스터 팔레트 약 48색** (차분한 웜톤). 시간대 변화는 팔레트 스왑이 아닌 씬 틴트+조명 레이어
- 아웃라인: 외곽만 다크브라운 계열(순검정 금지), 광원 좌상단 고정

### 제작 파이프라인

1. AI 초안 생성 (스타일 가이드 프롬프트)
2. 자동 정제 스크립트(Python): 팔레트 강제 양자화, 그리드 정렬, 배경 제거
3. 터치업 (픽셀 단위 수정)
4. **스타일 린터**: 팔레트 밖 색·크기 위반 자동 검출 (CI 포함)
5. 카탈로그 등록 (`items/*.json`)
6. 프리뷰 갤러리 웹페이지 (전 아이템 육안 검수)

### 캐릭터

베이스 바디 + 파츠 레이어 조합. 4방향 걷기·앉기·감정 애니메이션.
파츠 1종 추가 → 전 조합 자동 적용.

## 5. 사운드

- **BGM**: 시간대별 로파이 (아침 어쿠스틱→낮 로파이→노을 시티팝풍→밤 재즈). 크로스페이드 전환. 시즌1은 상업 사용 가능 무료 라이선스 음원, 추후 커스텀 교체.
- **앰비언트**: 공간별 사운드스케이프 (거리 웅성임 / 숲길 새·바람 / 포차 지글지글 / 비 오는 날 빗소리 우선).
- **SFX**: 재질별 발소리, 배치음, 코인, 이모트. 전부 낮고 부드러운 톤.

## 6. 아키텍처

### 클라이언트 (Phaser 3 + TS + Vite)

- Scenes: `Boot` → `Preload` → `Street`(공유 거리) / `Room`(개인 방) + HTML 오버레이 UI(채팅·인벤토리·상점·거래창)
- Systems: 이동/충돌, 꾸미기(배치), 인벤토리, 퀘스트, 거래, 시간/날씨, 오디오
- **`NetworkAdapter` 인터페이스**로 네트워킹 격리 → Supabase 구현체. 추후 전용 서버(Colyseus 등) 교체 지점.

### Supabase

- **Auth**: 카카오 OAuth (+구글 보조)
- **Realtime**:
  - presence: 접속자 목록
  - broadcast: 위치(10Hz)+클라이언트 보간, 이모트, 말풍선 — DB 미경유
  - postgres_changes: 방 배치·장터·게시판 변경 자동 반영
  - 채널: `street` 1개 + `room:{id}` (방 단위 트래픽 분리)
- **Postgres + RLS**: 본인 행만 쓰기, 공개 데이터는 모두 읽기
- **Edge Functions**: 구매/거래/보상 검증(치팅 방지), 날씨 프록시(키 은닉·10분 캐시), 일일 퀘스트 롤링

### 데이터 모델 (주요 테이블)

`profiles`(프로필·코인 잔액) · `characters`(파츠 조합) · `inventory` ·
`rooms`/`placements`(방·배치) · `trades`(1:1) · `listings`(장터) ·
`quests`/`quest_progress` · `guestbook` · `mail` · `wall_pixels`(협동 벽화) ·
`coin_ledger`(모든 재화 증감 원장 — 치팅·버그 추적)

### 저장소 & 배포

- GitHub 저장소: 코드 + 아이템 카탈로그(JSON) + 스프라이트 + `supabase/migrations`
- GitHub Actions CI: 타입체크 → 단위 테스트 → 아트 스타일 린터 → Vercel 자동 배포

## 7. 에러 처리

- **네트워크 끊김**: 지수 백오프 재접속 + DB 상태 재동기화. 끊긴 동안 "혼자 모드"로 계속 (게임 중단 없음).
- **꾸미기**: 낙관적 UI (즉시 반영 → 서버 거부 시 롤백).
- **거래·구매**: Edge Function 내 트랜잭션 + `coin_ledger` 기록. 동시 거래 충돌·아이템 복제 방지 (멱등 처리).
- **날씨 API 장애**: 마지막 캐시 값 유지, 캐시도 없으면 '맑음' 폴백.

## 8. 테스트

- **Vitest 단위 테스트**: 경제(코인 증감), 거래 로직, 배치 규칙, 퀘스트 진행 등 순수 게임 로직
- **Edge Function 테스트**: 거래 검증·보상 지급 시나리오
- **Playwright 스모크 테스트**: 로그인(모킹) → 이동 → 가구 배치 → 저장 확인
- **아트 스타일 린터**: CI에서 팔레트·크기 규칙 검증

## 9. 시즌1 범위 밖 (명시적 제외)

- 음성 대화 (시즌2 후보)
