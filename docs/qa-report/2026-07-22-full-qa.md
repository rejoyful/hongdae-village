## 수정 완료 (P0/P1)

### P0-1. `earn_activity` 퀘스트 와일드카드 → 무한 코인
`p_kind like 'quest\_%'` 조건 때문에 `quest_아무거나1`, `quest_아무거나2`… 임의 문자열마다
하루 40코인 지급 가능 (콘솔에서 RPC 직접 호출로 무한 파밍). → **5개 퀘스트 화이트리스트**로 교체.

### P0-2. 인벤토리 직접 쓰기 → 무한 코인 발행
`inv_insert_own/update_own` RLS가 본인 행의 **임의 수량** 쓰기를 허용
→ `qty=10000` 위조 후 `sell_item` 반값 환급 반복 = 코인 무한 발행.
→ 인벤토리·배치 테이블 직접 쓰기 권한 회수, 서버 RPC 3종 신설:
`grant_starter()`(스타터 지급) / `place_item()`(소유·수량·좌표 검증+차감 원자) /
`pickup_item()`(삭제+복귀 원자). 클라이언트는 RPC 우선 + 구 스키마 폴백(전환기 안전).

### P1-1. 배치 서버 검증 부재
좌표·아이템 보유 검증 없이 placements 직접 insert 가능(안 산 가구 배치, 좌표 오염, 행 스팸).
→ P0-2의 `place_item` RPC가 경계(1..10×1..8)·보유·방당 200개 상한까지 검증.

### P1-2. 채널 join 전 브로드캐스트가 REST로 유출
`send()`가 join 완료 전 REST 폴백으로 나가 위치·채팅 유실 및 rate limit 유발
(콘솔 경고 실측: "Realtime send() is automatically falling back to REST API").
→ `channel.state === 'joined'`일 때만 전송.

### P2 (동봉 수정). 일일 리셋이 UTC 기준
출석·활동 보상 하루 경계가 UTC(한국 오전 9시 리셋) → `kst_day_start()`로 KST 자정 기준.

## 운영 조치 필요 (사용자 실행)

2. **테스트 계정 정리** — 아래 SQL로 목록 확인 후 삭제 (랭킹 품질):

```sql
-- ① 삭제 후보 미리보기 (닉네임·코인·가입일)
select u.id, p.nickname, p.coins, u.created_at
from auth.users u join public.profiles p on p.id = u.id
where p.nickname in (
  '지속성테스트','경제테스트','연결테스트','워크테스터','모션테스터',
  'QA검증A','QA검증B','QA멀티A','QA멀티B','정밀A','정밀B','채널조사','QA배치검증','얍'
) or p.nickname like 'e2e%'
order by u.created_at;

-- ② 확인 후 삭제 (profiles·inventory·ledger는 cascade, rooms는 소유 해제)
delete from auth.users where id in (
  select u.id from auth.users u join public.profiles p on p.id = u.id
  where p.nickname in (
    '지속성테스트','경제테스트','연결테스트','워크테스터','모션테스터',
    'QA검증A','QA검증B','QA멀티A','QA멀티B','정밀A','정밀B','채널조사','QA배치검증','얍'
  ) or p.nickname like 'e2e%'
);

-- ③ 주인 없는 방에 남은 배치물 정리 (다음 입주자에게 남의 가구가 보이지 않게)
delete from public.placements where room_id in (
  select id from public.rooms where owner_id is null
);
```

> `얍`은 실유저일 가능성이 있으면 ①의 가입일·코인으로 판단 후 목록에서 빼세요.
